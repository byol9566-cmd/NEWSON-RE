# 뉴스온(NEWSON) 웹사이트 리뉴얼 PRD

> 본 문서는 현재 `newson_renew` 디렉토리의 정적 HTML/CSS/JS 사이트를 분석하여 도출한 제품 요구사항 정의서입니다. 리뉴얼 목표 스택은 **Next.js**입니다.

---

## 1. 제품 개요

### 1.1 서비스 정체성
- **회사명**: 뉴스온 / (주)뉴스온미디어 (NEWSON)
- **업종**: 언론홍보대행사 (News Marketing & Viral Marketing Company)
- **핵심 사업**: 보도자료 배포, 인터넷 기사광고, 언론보도, 바이럴 마케팅, 브랜드대상 인증
- **대표**: 이은별 / 사업자등록번호 428-86-00314
- **소재지**: 경기도 하남시 미사대로 550, C동 10층 1001호 브이17(덕풍동,현대지식산업센터 한강미사)
- **연락처**: 031-758-0215 / newsmarketing@daum.net / 운영 평일 09:00~18:00

### 1.2 핵심 가치 제안 (Value Proposition)
외신·방송사·통신사·일간지·주간지·월간지·온라인매체 등 **908개 언론사 / 2,000여 명 기자**에게 보도자료를 이메일로 발송. "비용 대비 가장 효율적인 언론홍보 마케팅" 포지셔닝. 당일 신속 처리 강조.

### 1.3 리뉴얼 동기 (현 사이트의 한계)
- 모든 핵심 콘텐츠가 외부 호스팅 이미지(`newson.co.kr`)로 박혀 있어 텍스트가 아님 → **SEO·접근성·유지보수 취약** (가격표·사업분야·브랜드대상 소개가 전부 JPG).
- UI디자인이 너무 구형이라 새롭게 디자인할 필요성.
- 네이버에서만 '뉴스온'이라는 검색어 중 상단에 뜨고 구글 같은 사이트에서는 아직 그 순위가 미약.
- 사용자가 본 이후 결제까지 이어지는 플로우가 부족함.
- 모바일에서의 UI까지 깔끔하게 나오는 반응형 웹이 필요.
- 로그인/회원가입/즐겨찾기/검색이 placeholder(`href="#"`)로만 존재 → **미구현 기능**.
- 공지/포트폴리오/Q&A 게시판이 정적 빈 테이블 → **CMS 부재**.
- 온라인 문의 폼이 `alert()`만 띄우고 실제 전송 없음 → **리드 수집 누락**.
- 페이지마다 헤더/푸터/사이드바 HTML이 복붙 중복 → **컴포넌트화 필요**.

## 4. 페이지별 기능 요구사항

### 4.1 홈 (`/`)
- 자동재생 히어로 슬라이더(3장, 4.5s, hover 시 정지), prev/next/dots 컨트롤.
- 인포 카드 4종: 공지사항(최근 N건), 온라인 문의 CTA, 비용/이용절차, 오시는 길.
- 공지 카드는 CMS/데이터 소스와 연동(현재 "등록된 공지사항이 없습니다" 빈 상태).

### 4.2 보도자료배포 (`/promotion/press-release`) — **콘텐츠 핵심 페이지**
- 실적 통계: 908개 매체 / 2,000+ 기자 / 당일 처리 (텍스트·애니메이션 카운터로 재구현).
- 요금 티어(텍스트 테이블로 재구현, 현재 일부만 노출):
  - 보도자료 배포 55,000원 / 마이너 110,000 / 중급 165,000 / 프리미엄 275,000 / 메이저 440,000 / TOP 메이저 660,000원
- 신청 프로세스 스텝(입금 → 자료 발송 → 검토/회신 → 배포).
- 결제 안내: 하나은행 260-910013-86604 (주)뉴스온미디어.

### 4.3 온라인문의 (`/inquiry`) — **리드 수집 핵심**
- 필드: 성명(필수), 전화(3분할 자동 포커스 이동), 이메일, 제목(필수), 설명(필수), 파일첨부.
- 개인정보 수집·이용 동의 체크박스(미동의 시 제출 차단).
- **신규 요구**: 실제 전송(API Route + 이메일/DB 저장), 스팸 방지, 성공/실패 상태 UI, 서버측 검증.

### 4.4 클라이언트 (`/clients`)
- 로고 그리드, 페이지네이션, 제목/내용 검색. 데이터 소스화(현재 하드코딩 8개).

### 4.5 고객센터 (`/customer`)
- 탭: 공지사항 / 포트폴리오 / Q&A. URL 쿼리(`?tab=`)와 동기화.
- 게시판 테이블 + 검색. CMS 또는 정적 데이터 소스 연동.

### 4.6 회사소개 (`/about/*`)
- 브랜드대상: 이미지 의존 콘텐츠를 텍스트/시맨틱 구조로 재작성.
- 오시는 길: 지도 임베드 + 주소/대중교통 안내.

### 4.7 공통 레이아웃
- 탑바 / 스티키 헤더(스크롤 시 그림자) / 글로벌 내비 / 모바일 햄버거 드로어 / 서브 히어로 슬라이더 / 좌측 사이드바(메뉴 + 상담안내 박스) / 브레드크럼 / 푸터.
- 사이드바 "상담안내" 박스(전화·운영시간·이메일·온라인문의 CTA)는 전 서브페이지 공통 컴포넌트.

## 5. 디자인 방향

### 5.1 현재 디자인 토큰 (style.css 기준 — 리뉴얼 기준선)
```
--primary:   #e8234a   (레드, 강조/포인트)
--accent:    #0072bc   (블루, 메인 액션/링크)
--accent-dk: #005a96
--accent-lt: #e8f4fc
배경: #f7f8fa / 텍스트: #333 / 카드 그림자, radius 8px
폰트: Malgun Gothic 계열 (→ 리뉴얼 시 웹폰트 검토: Pretendard 등)
컨테이너 1100px, 사이드바 200px, 반응형
```

### 5.2 리뉴얼 디자인 원칙
- 깔끔한 한국형 B2B 코퍼레이트 톤 유지, 신뢰감·전문성 강조.
- **레퍼런스 사이트(사용자 제공 예정) 기반으로 디자인 시스템 재정의** → 색/타이포/간격/컴포넌트 토큰화.
- 이미지로 박힌 콘텐츠를 컴포넌트(가격표, 스텝, 통계 카운터, 클라이언트 그리드)로 재구축.
- 모바일 퍼스트 반응형, 접근성(WCAG AA), 시맨틱 HTML.

> 디자인 레퍼런스는 사용자가 추후 제공. `design-reference-analyzer` 에이전트가 분석하여 디자인 시스템 명세를 산출하고, `nextjs-frontend-developer`가 구현, `visual-qa-engineer`가 검증한다.

## 6. 기술 스택 / 아키텍처

- **프레임워크**: Next.js (App Router, TypeScript 권장)
- **스타일**: CSS Modules 또는 Tailwind (디자인 토큰 single source) — 레퍼런스 분석 후 확정
- **컴포넌트**: Layout(Header/TopBar/MobileNav/Footer), SubPageShell(Sidebar+ConsultBox+Breadcrumb), HeroSlider, InfoCard, PricingTable, ProcessSteps, StatCounter, ClientGrid, BoardTable, InquiryForm
- **데이터**: 공지/포트폴리오/Q&A/클라이언트 → 초기엔 정적 콘텐츠(JSON/MDX), 차기 CMS 후보
- **문의 폼**: Route Handler(`/api/inquiry`) + 이메일 발송/DB 저장 + 검증 + 스팸 방지
- **이미지**: 외부 `newson.co.kr` 의존 제거, `next/image` + 로컬/CDN 자산으로 이전
- **SEO**: Metadata API, 시맨틱 마크업, sitemap/robots, OG 태그
- **MCP 도구**: Context7(Next.js 최신 문서 조회), Playwright(레퍼런스 캡처·시각 QA)
